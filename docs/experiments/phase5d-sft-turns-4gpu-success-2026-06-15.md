# Phase 5D Turn-Level 4-GPU SFT Warmup Success

Date: 2026-06-15

## Goal

Run a short turn-level SFT warmup that trains the model to emit exactly one
agent action per assistant turn. The key behavior is that `<observation>` is
provided as a runtime/user message, not generated by the assistant.

This run continues from the Phase 5C full-trace SFT checkpoint, using the
turn-level export from the 500-example gold-evidence synthetic set.

## Analysis

This run directly addresses the Phase 5C generation issue where the model learned
to emit full traces, including hallucinated `<observation>` blocks. The SFT
objective now separates assistant actions from environment observations in the
chat history, so the next inspection should test two behaviors separately:

- question-only prompt should produce a single `<search>...</search>` action
- question plus runtime `<observation>` should produce a single `<answer>...</answer>`

The low final validation loss means the model fit this small 500-example
turn-level dataset. It does not by itself prove rollout behavior is fixed; the
next step is to merge `global_step_40` into a Hugging Face checkpoint and run the
same generation inspection protocol used after Phase 5C.

## Next Steps

3. If inspection improves, use the merged checkpoint as the initial policy for
